# 📇 Функция распознавания визиток

## 🎯 Что это?

После того как пользователь поделился своим контактом через Telegram, бот предлагает сфотографировать визитную карточку. AI автоматически распознает все данные с визитки и сохраняет их в базу данных.

---

## 🚀 Как это работает?

### **Шаг 1: Поделиться контактом**
Пользователь выбирает:
- 📲 **Поделиться контактом** - Telegram автоматически передает номер телефона
- ✍️ **Написать свои контакты** - Пользователь вводит контакты текстом

### **Шаг 2: Предложение сфотографировать визитку**
После получения контакта бот предлагает:
```
📇 Дополнительно (необязательно):

Если у вас есть визитная карточка, сфотографируйте её!
Наш AI автоматически распознает все контакты и сохранит в базе.

Это поможет нам лучше понять ваш бизнес и связаться удобным способом.
```

Две кнопки:
- 📸 **Сфотографировать визитку**
- ⏭️ **Пропустить этот шаг**

### **Шаг 3: Отправка фото**
Пользователь отправляет фото визитки.

### **Шаг 4: AI распознавание**
Бот показывает: "📸 Фото получено! Анализирую визитку..."

AI анализирует изображение и извлекает:
- 👤 **Имя** - Полное имя с визитки
- 💼 **Должность** - Position/Title
- 🏢 **Компания** - Название компании
- 📱 **Телефон** - Все номера телефонов
- 📧 **Email** - Email адреса
- 🌐 **Сайт** - Веб-сайт компании
- 📍 **Адрес** - Физический адрес
- ℹ️ **Дополнительная информация** - Любые другие данные

### **Шаг 5: Подтверждение**
Бот показывает распознанные данные:
```
✅ Данные распознаны:

👤 Имя: Иван Иванов
💼 Должность: CEO
🏢 Компания: Tech Solutions Ltd.
📱 Телефон: +7 999 123-45-67
📧 Email: ivan@techsolutions.com
🌐 Сайт: www.techsolutions.com
```

## 💾 Что сохраняется в базу данных?

### **Таблица `contacts`**
Новые поля:
- `company` VARCHAR(500) - Название компании
- `position` VARCHAR(500) - Должность
- `website` VARCHAR(500) - Веб-сайт
- `address` TEXT - Адрес
- `business_card_data` JSONB - Полные данные в JSON формате

### **Преимущества JSONB**
Все данные с визитки сохраняются как в структурированном виде (отдельные поля), так и в полном JSON формате. Это позволяет:
- Сохранить любую нестандартную информацию
- Легко добавлять новые поля без миграции БД
- Быстрый поиск по JSON данным (PostgreSQL JSONB)

## 🤖 Технические детали

### **AI Vision API**
Используется:
- **Модель**: `gpt-4o` (OpenRouter)
- **API**: OpenRouter Vision API
- **Temperature**: 0.1 (для точного извлечения данных)
- **Формат ответа**: JSON

### **Промпт для AI**
```
Проанализируй эту визитку и извлеки все возможные контактные данные.

Верни ответ СТРОГО в формате JSON (без markdown, без ```json):
{
    "name": "Полное имя",
    "position": "Должность",
    "company": "Название компании",
    "phone": "Номер телефона",
    "email": "Email адрес",
    "website": "Веб-сайт",
    "address": "Адрес",
    "additional_info": "Любая дополнительная информация"
}

Если какое-то поле не найдено, укажи null.
```

### **Обработка изображения**
1. Пользователь отправляет фото визитки
2. Бот получает `file_id` самого большого размера фото
3. Получает `file_path` через `bot.get_file()`
4. Отправляет URL изображения в Vision API
5. AI анализирует и возвращает JSON
6. Данные сохраняются в базу

## 🔐 Безопасность

1. **Изображения не хранятся** - Обрабатываются через Telegram CDN
2. **JSONB в PostgreSQL** - Безопасное хранение структурированных данных
3. **Валидация данных** - AI извлекает только релевантную информацию
4. **Опциональный шаг** - Пользователь может пропустить сканирование
