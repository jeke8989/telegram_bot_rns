# 🎙️ Транскрибация голосовых сообщений

## Обзор

Neuro-Connector Bot теперь поддерживает **автоматическую транскрибацию голосовых сообщений**! Пользователи могут отвечать на любые вопросы бота голосом, и бот автоматически распознает речь, используя модель Whisper через OpenRouter API.

## Как это работает

### Пользовательский опыт:

1. **Бот задает вопрос** (например: "Какой процесс отнимает больше всего времени?")
2. **Пользователь отвечает голосовым сообщением** 🎤
3. **Бот обрабатывает аудио:**
   - Показывает индикатор "печатает..."
   - Отправляет сообщение "🎙️ Обрабатываю голосовое сообщение..."
   - Скачивает аудио файл
   - Отправляет на транскрибацию в OpenRouter API
4. **Бот показывает распознанный текст:**
   - ✅ Распознано: *"обработка заявок клиентов"*
   - Обрабатываю ваш ответ...
5. **Бот продолжает диалог** на основе распознанного текста

### Технический процесс:

```
┌─────────────────┐
│ Голосовое       │
│ сообщение       │
│ (Telegram)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Bot получает    │
│ файл (OGG)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Скачивание во   │
│ временный файл  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ OpenRouter API  │
│ Whisper-v3      │
│ Transcription   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Распознанный    │
│ текст           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Обработка как   │
│ текстовое       │
│ сообщение       │
└─────────────────┘
```

## Поддерживаемые форматы

Telegram отправляет голосовые сообщения в формате **OGG (Opus)**. OpenRouter API поддерживает следующие форматы:

- ✅ OGG (Opus) - формат Telegram по умолчанию
- ✅ MP3
- ✅ MP4
- ✅ MPEG
- ✅ M4A
- ✅ WAV
- ✅ WEBM

## Конфигурация

### Переменные окружения (.env):

```env

OPENROUTER_API_KEY=your_openrouter_api_key_here
OPENROUTER_MODEL=gpt-4o  # Модель для текстовой генерации

# Модель для транскрибации через OpenRouter настроена в коде:
# whisper_model = "openai/whisper-large-v3"
```

**ВАЖНО:** Мы используем ТОЛЬКО OpenRouter для всех AI-интеграций, включая транскрибацию голосовых сообщений. Отдельный ключ OpenAI НЕ требуется.

### Параметры API:

```python
# В ai_analyzer.py
self.whisper_model = "openai/whisper-large-v3"

# Timeout для транскрибации
timeout = aiohttp.ClientTimeout(total=60)  # 60 секунд
```

## Обработка ошибок

Бот корректно обрабатывает различные сценарии ошибок:

### 1. Ошибка API (нет доступа, превышен лимит):
```
❌ Извините, не удалось распознать аудио. Попробуйте позже.

Пожалуйста, попробуйте написать текстом.
```

### 2. Ошибка при скачивании файла:
```
❌ Произошла ошибка при обработке голосового сообщения.

Пожалуйста, попробуйте написать текстом.
```

### 3. Файл не найден:
```
❌ Ошибка: файл не найден

Пожалуйста, попробуйте написать текстом.
```

## Ограничения

### Telegram:
- **Максимальная длительность:** ~1 минута (ограничение Telegram для голосовых)
- **Формат:** OGG (Opus)
- **Размер:** до 1 MB (обычно)

### OpenRouter API:
- **Максимальный размер файла:** 25 MB
- **Таймаут:** 60 секунд
- **Rate limits:** зависят от вашего плана OpenRouter

## Стоимость

**Whisper Large V3** через OpenRouter:
- **Модель:** `openai/whisper-large-v3`
- **Цена:** ~$0.006 за минуту аудио
- **Качество:** Высокое (multilingual, 99+ языков)

## Поддержка

При возникновении проблем с транскрибацией:

1. **Проверьте API-ключ:** Убедитесь, что `OPENROUTER_API_KEY` корректный
2. **Проверьте баланс:** На вашем аккаунте OpenRouter достаточно средств
3. **Проверьте логи:** Посмотрите логи бота для деталей ошибки
4. **Проверьте соединение:** Убедитесь, что бот может подключиться к OpenRouter API
